## §1 — knowledge/page.tsx changes

### §1.A — Header: knowledge score as inline subtitle text [NEW — addresses auditor concern #1]

**Current code (lines 329-350):**
```tsx
<div className="flex flex-wrap items-center gap-4">
  <h1 className="font-heading text-xl font-bold text-charcoal md:text-2xl">{t('pageTitle')}</h1>
  {sufficiencyScore.data && (
    <div className="flex items-center gap-2">
      <span className="text-sm text-dune">{t('knowledgeScore')}</span>
      <span className="font-heading text-xl font-bold text-charcoal">
        {sufficiencyScore.data.score}/100
      </span>
      <span className={`text-sm font-medium ${...colorClass}`}>
        {gradeLabel}
      </span>
    </div>
  )}
</div>
```

**Problem:** Score and title are sibling flex children — not an inline subtitle. The score renders as a separate visual unit with its own label, bold number, and colored grade, rather than compact inline text in the header.

**Required change:** Restructure the header so the `<h1>` stands alone and the score appears as a compact `<p>` subtitle below it:

```tsx
<div className="space-y-0.5">
  <h1 className="font-heading text-xl font-bold text-charcoal md:text-2xl">
    {t('pageTitle')}
  </h1>
  {sufficiencyScore.data && (
    <p className="text-sm text-dune">
      {t('knowledgeScore')}:{' '}
      <span className="font-medium text-charcoal">
        {sufficiencyScore.data.score}/100
      </span>
      {' — '}
      <span className={`font-medium ${
        sufficiencyScore.data.score >= 80 ? 'text-teal' :
        sufficiencyScore.data.score >= 60 ? 'text-gold' :
        'text-sunset'
      }`}>
        {sufficiencyScore.data.score >= 80
          ? t('knowledgeScoreExcellent')
          : sufficiencyScore.data.score >= 60
          ? t('knowledgeScoreGood')
          : t('knowledgeScoreNeedsWork')}
      </span>
    </p>
  )}
</div>
```

This renders as: `Knowledge` (heading) / `Score: 72/100 — Good` (small subtitle text) — matching the spec format `"Knowledge · Score: 72/100 — Good"` in a compact, always-visible layout.

**Remove from imports:** `CheckCircle2` from `lucide-react` if it's only used in the learnings section (verify before removing).

**Actions cell implementation:**
```tsx

  {doc.title ? (
    deleteConfirm === doc.title ? (
      /* existing inline delete-confirm row — unchanged */
      <span className="flex items-center justify-end gap-2">...</span>
    ) : (
      <>
        <Button
          size="sm"
          variant="ghost"

            setOpenDocMenu((prev) => (prev === doc.title ? null : doc.title!))
          }
        >
          <MoreHorizontal className="h-4 w-4" />
        </Button>

          <div

              type="button"
```
